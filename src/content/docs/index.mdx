## 先选路，再动手

本站有两条**互不混用命令**的零基础闭环：Claude Code 与 Codex。公共知识（终端、Git、验证思路）放在方法论与附录；工具专属命令各自独立。

### 30 秒决策

| 你的情况                                    | 建议路径                                                    | 第一站                                                               |
| ------------------------------------------- | ----------------------------------------------------------- | -------------------------------------------------------------------- |
| 想跟 Anthropic 工具从零做出可展示页面       | [AI 编程零基础入门](/paths/#ai-coding-zero)（约 6–8 小时）  | [开始之前](/claude-code/preflight/)                                  |
| 想跟 OpenAI Codex 做出可公开访问的小站      | [Codex 零基础完整路线](/paths/#codex-zero)（约 4–6 小时）   | [安装、登录与成本](/codex/quickstart/)                               |
| 已会基础对话，想提效（记忆 / Skill / 节奏） | [日常效率提升](/paths/#daily-efficiency)（约 4–6 小时）     | [学习路径总览](/paths/#daily-efficiency)                             |
| 要 Hook / MCP / 子 Agent / 配置即代码       | [高级定制与自动化](/paths/#advanced-custom)（约 7–10 小时） | [学习路径总览](/paths/#advanced-custom)                              |
| 还没装过终端 / 不懂 Git                     | 先补预备知识                                                | [通用环境](/methodology/basics/) → [Git 速成](/appendix/git-basics/) |
| 报错卡住了                                  | 先排障，再回路径                                            | [常见问题排查](/appendix/troubleshooting/)                           |

## 卡住了怎么办

1. 先看 [常见问题排查](/appendix/troubleshooting/)（安装、PATH、权限、发布失败）。
2. Codex 专属问题看 [Codex 故障排查](/codex/troubleshooting/)。
3. 仍解决不了：到 [GitHub Discussions](https://github.com/estelledc/zero-to-ai/discussions) 贴出命令、报错原文和你走的路径名。

本站教程由 [Jason](https://github.com/estelledc) 维护。教程结构与类比可由 AI 辅助生成；时效性命令和配置的复核范围、日期与未完成项记录在官方资料验证矩阵中。
