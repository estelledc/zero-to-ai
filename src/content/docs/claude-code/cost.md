## 为什么这篇很重要

大多数 Claude Code 教程不提钱。但最大的用户震惊不是功能学不会——是月底看到账单。

这篇文章确保你在花钱之前就知道钱花在哪。读完你会理解 token 是什么、三种模型的成本差异、怎么估算自己的月账单、以及五条可操作的成本控制方法。

## Token 是什么（30 秒版）

**类比：出租车计价器。**

出租车不是按“到达目的地”一次收费，是按里程——每跑一段，计价器跳一下。Token 就是 Claude 的“计价里程”：每处理一小段文字，跳一个 token。

- 1 token 约等于 0.75 个英文单词，或约 0.5 个中文字
- **Input token**（输入）：Claude 读代码、读你的问题、重新读对话历史——“理解”环节
- **Output token**（输出）：Claude 写代码、写回复——“干活”环节
- Output token 的价格通常是 input token 的 2-5 倍

**具体例子**：

- 一个 1000 行的代码文件 ≈ 约 15,000 token
- 一次简短的问答（“这个报错是什么意思”）≈ 500-2,000 token
- 一次 2 小时的深度对话 ≈ 50,000-150,000 token

**关键洞见**：Claude 每轮对话都要把你之前所有的对话历史重新读一遍——这是它“记住上下文”的方式。所以对话越长，每轮的 input token 消耗越大。这就是为什么教程一直强调控制上下文——不只是因为 Claude 会变笨，也因为每多聊一轮，成本都在涨。

## 三种模型，三种价位

Claude Code 支持三种模型，价格差异很大。选择哪个模型是对成本影响最大的单一决策。

| 模型   | 定位     | Input ($/M tok) | Output ($/M tok) | 适合                            |
| ------ | -------- | --------------- | ---------------- | ------------------------------- |
| Haiku  | 快而便宜 | ~$0.80          | ~$4              | 简单任务、格式转换、grep 类操作 |
| Sonnet | 均衡     | ~$3             | ~$15             | 日常编程、代码审查、文档生成    |
| Opus   | 强而贵   | ~$15            | ~$75             | 复杂架构、多文件重构、深度推理  |

> 以上为近似价格，价格可能随时调整，请以 [Anthropic 官方最新定价](https://www.anthropic.com/pricing) 为准。M token = 百万 token。

**三种模型的实际使用场景**：

- Haiku：“这个文件里有哪些函数？” “把这段 JSON 格式化成多行” “帮我找一个变量名在哪里定义”——简单、快速、不需要深度推理
- Sonnet：“帮我重构这个 React 组件，把业务逻辑抽到 custom hook” “review 这个 PR 的代码” “帮我写一个完整的表单页面”——日常开发主力
- Opus：“设计这个电商系统的数据库 schema，考虑扩展性” “帮我分析这个分布式系统的瓶颈” “重构这个 3000 行的模块”——需要深度推理、多文件理解

## Opus 悖论：为什么最贵的模型可能更便宜

**乍一看**：Opus 的 unit price 大约是 Sonnet 的 5 倍。用 Opus 当然更贵。

**但算一下迭代次数**（以下为假设的单次任务总费用，含 input + output，不是每百万 token 单价）：

- 假设用 Sonnet 完成一次复杂任务的总费用约 $3：$3 x 3 轮迭代 = $9（假设 3 次才做对）
- 假设用 Opus 完成同一个任务的总费用约 $15：$15 x 1 轮 = $15

如果 Sonnet 一次就能做对：$3 < $15，Sonnet 更省钱。
如果 Sonnet 需要 6 轮才做对：$18 > $15，Opus 反而更省钱。

**这不是说 Opus 一定更便宜。而是说“单价便宜不等于总成本低”。** 选模型的原则：

- 你很清楚要做什么，任务简单明确 → Haiku 或 Sonnet（一次能做对的信心很高）
- 你不太确定怎么拆解，需要探索 → Sonnet（平衡推理能力和成本）
- 这是一个需要深度推理的架构级问题 → Opus（第一次做对的价值远大于单价差异）

**一句话口诀**：简单任务用便宜模型（省下来的钱确定），复杂任务用强模型（避免反复重来的成本不确定）。

**速查**：不确定用哪个模型？先用 Sonnet。大部分人 90% 的时间用 Sonnet 就够了，遇到特别简单的任务切 Haiku，遇到架构级难题切 Opus。

## 三段成本，你属于哪一段

和 [10 分钟上手](/claude-code/quickstart/) 中的成本概览一致，这里展开每个档位的典型画像。

### 轻量：$5-15 / 月

- 每天 20-40 分钟，周末基本不用
- 任务类型：小 bug 修复、代码解释、技术问题咨询
- 主力模型：Haiku 为主，偶尔 Sonnet
- 类比：每月一杯精品咖啡

### 中度：$50-100 / 月

- 每天 1-3 小时
- 任务类型：日常开发主力工具，新功能开发、代码审查、调试
- 主力模型：Sonnet 为主
- 类比：每月买一本技术书（但互动性远超一本书）

### 重度：$300+ / 月

- 全天候使用
- 任务类型：多项目并行、子 agent 委派、MCP 工具集成
- 主力模型：Opus 为主
- 类比：雇了一个全职初级开发（但 Claude 更快，且不需要 onboarding）

**怎么知道自己在哪一段**：用一周，每天结束前输入 `/usage` 记下数字。一周后取日均 x 30，就是你的月估算。不需要猜——数据会告诉你。

## 怎么控制成本（5 条可操作建议）

### 1. 用 /usage 常看

每次会话结束前输入 `/usage`，看一眼花了多少。和记账一样——不记不知道，记了才有概念。养成这个习惯只需要一周。

### 2. 简单任务切 Haiku

在 settings.json 中设置默认模型：

```json
"model": "claude-haiku-4-5"
```

或者在启动时指定：`claude --model claude-haiku-4-5`。

不是所有任务都需要 Opus。查文档、格式化代码、找函数定义——用 Haiku 就够了。

### 3. Opus 用完切回来

不要一直用最贵的模型。复杂任务开 Opus，做完了切回 Sonnet 或 Haiku。就像你不会开着跑车去买菜——工具匹配任务。

### 4. 遵守 2 小时规则

单会话不超过 2 小时。详情见[上下文管理](/claude-code/context/)。

不只是为了上下文质量——也为了成本。一个 2 小时对话的每轮 input token 消耗可能是新对话的 5-10 倍，因为每次都要重读全部历史。

### 5. MCP 按需启用，不要装一堆

每装一个 MCP 工具会占用 7-15% 的上下文空间（取决于工具的定义大小）。上下文大了 → 每轮 token 消耗多了 → 成本上升。只用你真正需要的 MCP。

## 为什么花 $300+ / 月可能是合理的

不是说你应该花这么多——是给你一个计算框架。

- **节省的时间**：每天 Claude 替你做了原本要 3-4 小时的事。比如：查文档、调试、写样板代码、格式化
- **学习加速**：遇到问题立刻有解释 vs 搜 StackOverflow 20 分钟、看 3 篇文章、还不知道哪个是对的
- **质量提升**：自动检查低级错误 vs 部署后才发现、回滚、修、再部署

如果 AI 每月帮你省 40 小时，$300 / 40 = $7.5 / 小时。这是你自己的时间单价。值不值是你自己判断——但现在你有数字可以算了。

> 价格随时可能调整。在做预算前，建议查看 [Anthropic 官方定价页](https://www.anthropic.com/pricing) 获取最新数据。

如果你想先零成本体验 Claude Code 的工作方式，可以通过第三方免费模型实现——详见[零成本 vibe coding](/claude-code/agnes-free-vibe-coding/)。

## 常见坑

1. **装了就一直用最贵的模型**：Opus 做简单任务 = 浪费。切模型只需要改一行配置
2. **从不看 /usage**：月底看到账单才震惊。每天看一眼，5 秒钟的习惯
3. **对话拖太长**：一个 session 聊 3-4 小时，后面 1 小时的每轮 input cost 是新对话的 10 倍
4. **以为 Claude Code 免费**：它不是。从第一天起就有成本。API key 不是你付了月费就无限用的——每次对话都在消耗
