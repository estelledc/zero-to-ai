## 这个教程适合谁

先花几分钟确认你是不是来对了地方。

1. **你能打开终端吗？** 能——继续。不能——先去[通用环境基础设施](/methodology/basics/)花 10 分钟，再回来。
2. **你有 GitHub 账号吗？** 有——继续。没有——去 [github.com/signup](https://github.com/signup) 注册一个（2 分钟，免费）。教程后面 `gh` CLI 和 Git 操作会用到。
3. **你能接受"AI 第一次输出大概率不对"吗？** 这是最重要的心态。把 Claude 想象成一个非常聪明但完全不了解你项目的新同事。他第一天上班，还没读过你的文档、不知道你的命名习惯、不清楚你讨厌哪种代码风格。他第一次写的代码大概率需要你告诉他哪里不对、重新来。这不是 bug，这是工作方式。Claude 需要你的反馈来校准，和你需要 mentor 给你 code review 反馈是一个道理。
4. **你愿意为这个工具花钱吗？（$5-300+/月）** Claude Code 不是免费的。轻量用户 $5-15，中度用户 $50-100，重度用户 $300+。详细说明见下方[需要花多少钱](#需要花多少钱)和 [成本完整指南](/claude-code/cost/)。

以上 4 条全中？继续往下。有一条不满足？先去补那条，再回来——页面在，不跑。

## 心理模型：10 秒理解 AI 编程

在你动手安装之前，先有一个画面——它是理解后面所有教程的基础。

你和 Claude 的每次对话，相当于在**一张办公桌上摆纸**。

- 桌子上放的纸 = 对话内容（你说的话、Claude 的回复、它读过的文件、它写的代码）
- 桌子的大小 = 上下文窗口（有限的——不同模型版本大小不同，但都是有限的）
- Claude 能看到桌上所有纸。但桌子满了，就只能在桌上挤——最早放上去的纸被挤到地上，再也找不回来。

这就解释了教程后面会反复出现的两个命令：

- **`/clear`** = 把桌上所有纸收走，换一张空桌面（切换任务时用）
- **`/compact`** = 把一叠纸的内容摘要成一张索引卡，节省空间但保留关键信息（任务还没做完但桌面快满了时用）

现在不用完全理解——教程里有[一整章](/claude-code/context/)讲上下文管理。先有这个画面就够了。

## 学完你能做什么

教程按三条学习路径组织，每条路径的目标不一样：

**学完基础路径后：** 用 Claude Code 写小页面、改 bug、理解陌生代码。不再对着报错信息干瞪眼——把报错贴给 Claude，它告诉你哪里错了、怎么修。

**学完效率路径后：** Claude 记住你的偏好（CLAUDE.md），换项目不用重新自我介绍。日常开发有固定节奏：新任务先 `/clear`，长任务主动 `/compact`，做完用[验证方法论](/claude-code/verify/)验收。不再需要手动重复配置、手动检查低级错误、手动在工具间切换。

**学完高级路径后：** Claude 自动触发检查（Hook）、多个子 Agent 并行工作、配置可以跨机器同步。你不再是一个人——你有一个能自动执行重复工作的小团队。

一句话总结学到的东西：不是"成为 10x 程序员"，而是"不让重复的机械操作吃掉你的注意力"。

## 需要花多少钱

具体数字取决于你的模型选择和用量。以下是 2026 年 6 月的参考范围：

**轻量用户：$5-15 / 月**
每天用 20-40 分钟，问技术问题、改小 bug、理解一段陌生代码。主力模型：Haiku（便宜，快，适合简单任务）。类比：每月一杯精品咖啡。

**中度用户：$50-100 / 月**
每天 1-3 小时，日常开发主力。Claude Code 帮你写功能、调试、重构代码、管理 git。主力模型：Sonnet（平衡速度和质量）。类比：每月一本技术书。

**重度用户：$300+ / 月**
全天使用，多项目、子 Agent、MCP 集成。主力模型：Opus（最强，最贵，适合复杂架构决策和棘手 bug）。类比：雇了一个全职初级开发（但 Claude 更快，不用倒水、不用开会）。

具体怎么算的、不同模型怎么选、怎么控制开销——详见[成本完整指南](/claude-code/cost/)。别担心，先学会用，再学会省。初学前几次对话的费用通常不到 $1。
