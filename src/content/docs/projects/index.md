前面的教程教你怎么用工具、怎么设计方法论。这个板块展示的是：**把工具和方法论结合起来，能做出什么真实的东西。**

这里有两类页面，请先分清再点进去：

| 类型                       | 适合谁                                 | 读完你应得到                                             |
| -------------------------- | -------------------------------------- | -------------------------------------------------------- |
| **Capstone（可交付作品）** | 零基础路径走到「有本地页面」之后       | 公开 URL、Git 历史、验收截图                             |
| **方法论实验报告**         | 已有 AI 编程助手、想系统记学习过程的人 | 对假设、边界与代价的诚实理解（不是 polished 产品说明书） |

## 怎么选

| 你的目标                                  | 去哪                                                                      | 前置                                                       |
| ----------------------------------------- | ------------------------------------------------------------------------- | ---------------------------------------------------------- |
| 把第一张个人页挂到网上（Claude 路径）     | [把第一张个人页面发布成小站](publish-first-site)                          | 已完成 [第一个页面](/claude-code/first-page/) 或等价本地页 |
| 用 Codex 交付公开小站                     | [Codex Capstone](/codex/capstone/)（在 Codex 专区，不在本列表重复教命令） | 已完成 Codex 前序：规则 → 修改/验证/Git                    |
| 了解 Learn Journal 这套记录方法是否适合你 | [Learn Journal](learn-journal)                                            | 能用终端 + 已有 Claude Code / 兼容适配器；详见实验页前提   |

> **不是 demo 秀场**
>
> Learn Journal 系列以「方法论实验报告」为主——记录设计决策、验证结果和已知局限。核心假设仍带 N=1 标签，可迁移性未承诺。

## Checkpoint

- □ 你能区分「要公开 URL 的 capstone」和「要理解边界的实验报告」
- □ 你已根据上表选好下一篇，而不是把 Learn Journal 当成零基础第一课
- □ 若目标是上线，你知道 Claude 路径用 publish-first-site，Codex 路径用 codex/capstone
