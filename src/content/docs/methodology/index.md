这些内容不绑定任何一个具体工具，而是讲通用思想：
怎么写出好的 AI 指令、怎么设计记忆系统、怎么编排多步骤工作流。

建议先学完至少一个工具的快速入门（比如 [Claude Code 10 分钟上手](/claude-code/quickstart/)），再来读方法论。

> **跟零基础路径？** 你只需要读下面第一篇「通用环境基础设施」，其余篇目是进阶内容，等你走完零基础路径后再回来。

- [通用环境基础设施](basics) — 终端、PATH、包管理器、环境变量一次性讲清楚
- [CLAUDE.md 编写哲学](claude-md-philosophy) — 为什么 CLAUDE.md 是 Claude Code 的核心、怎么写出好的项目指令
- [记忆系统设计原则](memory-system-design) — 为什么 AI 编程需要记忆系统、怎么设计可持续的知识管理流程
- [工作流编排思路](workflow-design) — 怎么把多个 AI 工具和 Skill 组织成高效的工作流
- [学习管理系统](learning-management) — 工件选择启发式、笔记管线、如何把对话变成永久知识
- [学习日志 Skill Pack](skill-pack) — 一键部署的结构化学习系统，下载即用
- [系统提示解剖学](prompt-anatomy) — 从泄露的真实 AI 系统提示中，学习工业级的指令设计模式
- [Skill 工程化设计](skill-engineering) — 从开源 Skill 项目中学习如何把提示词升级为可安装、可版本化的工程资产
