# 质量门禁

确保学习记录的质量，避免"写了但没用"的无效笔记。

---

## 核心规则

### R1：学习笔记必须有"来源"

`learnings/*.md` 的 front-matter 必须包含 `来源:` 字段。

来源可以是：

- 文章/书籍/课程名称
- "实践中发现" / "排查 bug 时学到"
- "同事 XX 教的" / "mentor 讲解"
- 对应的 `sources/` 文件链接

**为什么**：没有来源的知识无法追溯、无法验证、无法深入。

### R2：必须用自己的话

学习笔记的"核心要点"段不能是纯复制粘贴。AI 在创建笔记时应该引导用户用自己的话表达。

**判断方法**：如果内容读起来像教科书/官方文档的原文，提醒用户"能用你自己的话说说吗？"

### R3：必须有 why

每篇学习笔记必须回答"为什么重要"或"什么场景下用"。纯粹的 what（是什么）不够。

### R4：不重复创建

创建新笔记前，先检查 `learnings/` 目录是否已有相关内容：

- grep 关键词
- 如果已有，建议补充而不是新建
- 如果角度不同（比如已有"Redis 基础"，新的是"Redis 持久化"），可以新建但加交叉引用

### R5：交叉引用

新笔记如果跟已有笔记有关联，添加相对链接：

```markdown
## 关联

- [Redis 基础](redis-basics.md) — 本文是其深入
- [数据库持久化对比](database-persistence.md) — 更广的视角
```

---

## 启用时机

**第一周不启用质量门禁**。新用户先跑起来，形成记录习惯比质量更重要。

**第二周开始**，AI 在创建笔记时自动检查以上规则。如果不满足，温和提醒而不是阻止：

```
AI：笔记创建好了。不过我注意到你没有写"为什么重要"——
    加上这个会让笔记以后更有用。要补充一下吗？
```

---

## 可选增强：lint 脚本

如果用户使用 git，可以把 `scripts/lint-notes.py` 设为 pre-commit hook，自动检查 R1（来源字段）和死链。

这是可选的，不是必须的。
