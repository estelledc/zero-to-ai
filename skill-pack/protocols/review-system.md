# 间隔复习系统

对抗遗忘曲线，帮助用户长期记住学过的东西。

---

## 启用条件

当 `learnings/` 目录中有 **≥ 5 篇**笔记时自动启用。之前不提复习（没有足够内容可复习）。

---

## 掌握度追踪

每篇 learning 的 front-matter 包含两个字段：

```yaml
掌握度: 0.3
上次复习: 2026-06-17
```

- `掌握度`：0.0（完全不会）到 1.0（完全掌握），初始值 0.3
- `上次复习`：最近一次复习的日期，初始值为创建日期

---

## 复习优先级

```
priority = (1 - 掌握度) × (距今天数 + 1)
```

priority 越高，越应该复习。

示例：

- 掌握度 0.3，10 天前学的 → priority = 0.7 × 11 = 7.7（高，该复习了）
- 掌握度 0.8，3 天前学的 → priority = 0.2 × 4 = 0.8（低，不急）

---

## 复习流程

### 1. 选择复习内容

AI 扫描 `learnings/` 目录，读取 front-matter，计算 priority，选出 top 3。

### 2. 苏格拉底式验证

对每篇笔记，不展示内容，直接问用户：

```
AI：不看笔记，你能说说 [概念名] 的核心要点吗？
```

### 3. 评估回答

| 回答质量              | 掌握度变化 | 下次复习间隔           |
| --------------------- | ---------- | ---------------------- |
| 完全正确 + 能举例应用 | +0.15      | 拉长（掌握度 × 14 天） |
| 大方向对但细节模糊    | +0.05      | 保持（7 天）           |
| 说不出来 / 需要提示   | -0.10      | 缩短（3 天）           |
| 有明显误解            | -0.15      | 立即重新讲解           |

### 4. 更新状态

修改对应 learning 文件的 front-matter：

```yaml
掌握度: 0.45 # 更新后的值
上次复习: 2026-06-17 # 今天的日期
```

### 5. 误解处理

如果用户有明显误解，触发六步讲解协议（参考 `protocols/explain-protocol.md`）重新讲解该概念，然后更新笔记中的"常见误区"段。

---

## 触发时机

| 场景                   | 行为                                                  |
| ---------------------- | ----------------------------------------------------- |
| 用户说"复习"           | 立即开始复习流程                                      |
| 日终回顾时             | 如果有 priority > 5 的笔记，提醒"有 N 个概念该复习了" |
| 用户说"不知道做什么"   | 推荐复习 priority 最高的内容                          |
| 用户主动问某个已学概念 | 视为一次复习机会，更新掌握度                          |

---

## 设计原则

- **不增加负担**：复习是建议不是强制，用户可以说"今天不想复习"
- **不是死记硬背**：验证的是理解和应用能力，不是背诵
- **保留上下文**：复习时可以展示笔记的关联内容，帮助回忆
- **正向反馈**：掌握度提升时告诉用户"你的 Redis 掌握度从 0.4 提升到 0.55 了"
