# 学习导航 Skill

> 帮用户找到下一步该做什么：复习旧知识或探索新方向。

## 触发条件

- "不知道做什么" / "下一步" / "学习进度"
- "复习" / "review"
- "学了什么" / "这周学了什么"

## 模式识别

| 用户信号                  | 模式 | 行动                         |
| ------------------------- | ---- | ---------------------------- |
| "不知道做什么" / "下一步" | 推荐 | 基于掌握度推荐复习或新方向   |
| "复习" / "review"         | 复习 | 执行间隔复习流程             |
| "学了什么" / "进度"       | 状态 | 展示知识覆盖概览             |
| "这周学了什么"            | 总结 | 扫描本周 daily/ + learnings/ |

---

## 推荐模式

### 流程

1. 扫描 `learnings/` 目录，读取每篇的 front-matter（掌握度、上次复习）
2. 计算复习优先级（参考 `protocols/review-system.md`）
3. 判断：
   - 如果有 priority > 5 的笔记 → 推荐复习（"你有几篇笔记该复习了"）
   - 如果没有 → 推荐探索新方向

### 推荐新方向的依据

- 读取 `config.yaml` 中的学习方向
- 查看 `daily/` 最近几天提到的"深入挖掘"
- 查看 `sources/` 中状态为"未处理"的材料

### 输出格式

```
📋 学习状态：
- 已记录 X 篇学习笔记
- 其中 Y 篇需要复习（掌握度 < 0.5 或超过 7 天未复习）

建议下一步：
1. [复习] <笔记标题> — 上次复习 N 天前，掌握度 0.X
2. [新学] <方向> — 来自你的"深入挖掘"清单
3. [消化] <来源标题> — 还没看完的学习材料
```

## 复习模式

执行 `protocols/review-system.md` 中定义的完整流程：

1. 找出 priority 最高的 3 篇笔记
2. 逐篇进行苏格拉底式提问（不给看笔记内容）：
   - "X 是什么？用自己的话说说"
   - "它解决什么问题？"
   - "给我举个使用场景"
3. 根据回答质量更新掌握度：
   - 完全正确 + 能举例 → 掌握度 +0.15
   - 大方向对但细节模糊 → 掌握度 +0.05
   - 说不出来 → 掌握度 -0.1
4. 更新笔记 front-matter 中的 `掌握度` 和 `上次复习` 字段
5. 如果掌握度 < 0.3，建议重新学习（用六步讲解法）

## 状态模式

扫描 `learnings/` 目录，输出概览：

```
知识库概览：
- 总计 X 篇学习笔记
- 分类分布：frontend Y 篇 / backend Z 篇 / ...
- 掌握度分布：高(>0.7) A 篇 / 中(0.4-0.7) B 篇 / 低(<0.4) C 篇
- 最近一周新增：D 篇
```

---

## 总结模式

扫描本周的 `daily/` 和 `learnings/`，生成周总结：

```
本周学习总结（MM/DD - MM/DD）：
- 工作日 X 天，记录了 Y 篇日报
- 新增学习笔记 Z 篇：<列出标题>
- 解决问题 W 个：<列出标题>
- 掌握度提升最多：<笔记标题>（0.X → 0.Y）
```

## 不做的事

- 不在用户没有足够笔记时强推复习（< 5 篇时只推荐新学习）
- 不给出模糊建议（"继续加油"）——必须具体到某篇笔记或某个方向
- 不替用户决定学什么——给选项让用户选
