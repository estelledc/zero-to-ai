# 知识捕获 Skill

> 对话中识别学习内容，引导用户结构化记录。

## 触发条件

用户在对话中表达了学习内容：

- "今天学了 X" / "搞懂了 X" / "原来 X 是这样"
- 解释某概念超过 5 句话
- 解决了一个非平凡问题后
- 用户主动说"记一下" / "这个值得记"

## 执行流程

### 1. 判断工件类型

读取 `protocols/artifact-routing.md`，按规则判断应该记到哪里。

### 2. 去重检查

在对应目录中搜索是否已有相关笔记：

```
搜索 learnings/ 目录中是否有相同主题的文件
搜索 problems/ 目录中是否有相同问题的记录
```

- 如果已有 → 建议补充或更新已有笔记，而不是新建
- 如果没有 → 继续创建

### 3. 引导表达

不要直接帮用户写笔记。先引导用户用自己的话说：

- "你能用自己的话说说 X 是什么吗？"
- "它解决什么问题？没有它会怎样？"
- "你觉得什么场景下会用到？"
- "有没有踩过什么坑？"

如果用户已经在对话中充分表达了（能回答上述问题），可以跳过引导直接整理。

### 4. 创建笔记

根据工件类型选择模板：

| 类型     | 模板                    | 目录         | 命名规则                              |
| -------- | ----------------------- | ------------ | ------------------------------------- |
| 学习笔记 | `templates/learning.md` | `learnings/` | kebab-case，如 `react-useeffect.md`   |
| 问题记录 | `templates/problem.md`  | `problems/`  | kebab-case，如 `cors-blocked-api.md`  |
| 学习材料 | `templates/source.md`   | `sources/`   | kebab-case，如 `mdn-flexbox-guide.md` |
| 日常记录 | 追加到当天 `daily/`     | `daily/`     | `YYYY-MM-DD.md`                       |

### 5. 填充内容

- 用用户自己的话填写，不是 AI 的总结
- 必须填写"来源"字段（从哪里学到的）
- 必须填写"为什么重要"（不只是 what，要有 why）
- 添加交叉引用（关联已有笔记）

### 6. 确认

创建完成后告诉用户：

- 文件路径
- 建议后续：是否要补充代码示例、自测问题等

## 质量门禁

创建的笔记必须满足（参考 `protocols/quality-gates.md`）：

- R1: 有"来源"字段
- R2: 有"为什么重要"段落
- R3: 用自己的话（不是复制粘贴原文）
- R4: 文件名 kebab-case
- R5: 无重复（已做去重检查）

## 不做的事

- 不在用户没表达理解时强行创建笔记
- 不替用户写"是什么"段落（必须是用户的话）
- 不创建空模板让用户自己填（要么引导到能填，要么不创建）
