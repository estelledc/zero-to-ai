# 日终回顾 Skill

> 一天结束时，引导用户回顾并记录当天的工作和学习。

## 触发条件

- "下班了" / "收工" / "今天就到这"
- "写日报" / "记录今天"
- 一天的对话即将结束时

## 执行流程

### 1. 检查今日日报

检查 `daily/` 目录下是否已有今天的文件（`YYYY-MM-DD.md`）：

- 如果已有 → 读取内容，看哪些部分还是空的，引导补充
- 如果没有 → 使用 `templates/daily.md` 创建

### 2. 引导回顾

按以下顺序引导（已在对话中提到的可跳过）：

**工作内容**：

- "今天主要做了什么？"
- "完成了哪些任务？"

**困难和问题**：

- "遇到什么卡住的地方了吗？"
- "有什么花了比较长时间的事情？"

**学习收获**：

- "今天有学到什么新东西吗？"
- "有什么'原来如此'的时刻吗？"

**明日计划**：

- "明天打算做什么？"
- "有什么需要跟进的事情？"

### 3. 结构化记录

将用户的回答填入日报模板对应位置：

- `今日工作` → 任务清单（checkbox 格式）
- `遇到的困难` → 简述卡住的地方
- `今日学到` → 表格形式，每条一句话
- `深入挖掘` → 今天冒出来但没时间展开的问题
- `明日计划` → 具体可执行的步骤

### 4. 触发知识捕获

如果用户在回顾中提到了新概念或重要收获：

- 判断是否值得单独记录（参考 `protocols/artifact-routing.md`）
- 如果值得 → 提示"这个要不要单独记一篇学习笔记？"
- 用户同意 → 触发 `skills/capture/SKILL.md` 流程

### 5. 复习提醒

检查是否有待复习的笔记（参考 `protocols/review-system.md`）：

- 如果有 priority > 5 的笔记 → "你有 X 篇笔记该复习了，明天要不要花 10 分钟复习一下？"
- 如果没有 → 跳过

### 6. 生成总结

在日报末尾填写"一句话总结"——用一句话概括今天最重要的事。

## 输出示例

```
今天的日报已经写好了：daily/2025-01-15.md

今天你完成了 3 个任务，学到了 React useEffect 的清理机制。
明天计划：完成登录页面 + 复习 2 篇笔记。

有一个值得深入的点：为什么 useEffect 的依赖数组是浅比较？
要不要现在记一篇学习笔记？
```

## 质量要求

- "明日计划"必须具体可执行（不接受"继续学 X"）
- "今日学到"必须有 why/how（不只是"学了 X"）
- 如果用户说"没学到什么"，追问"有没有什么做法和昨天不一样的？"

## 不做的事

- 不自动扫描对话历史来填日报（各平台格式不同，且可能遗漏）
- 不统计 token 消耗（各平台 API 不同）
- 不生成 HTML 渲染版本（MVP 不需要）
- 不强制用户写日报（如果用户说"今天没什么"，尊重但建议至少记一句话）
