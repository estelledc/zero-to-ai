# 知识库健康检查 Skill

> 定期检查学习空间的质量和完整性，发现问题并建议修复。

## 触发条件

- "检查" / "整理" / "知识库状态"
- "有什么需要补充的吗"
- 用户使用超过一周后，主动提醒一次

## 前置条件

此 skill 在用户积累了一定量的笔记后才有意义：

- `learnings/` 中至少有 5 篇笔记
- 使用超过 5 天

如果不满足，告诉用户"现在笔记还不多，先专注记录，过几天再来检查"。

## 检查项目

### 1. 孤儿笔记

扫描 `learnings/` 中没有被任何其他文件引用的笔记：

- 检查方法：grep 所有 `.md` 文件中的链接，找出从未被链接到的 learnings
- 建议：为孤儿笔记添加"关联"段落，连接到相关笔记

### 2. 空壳笔记

扫描 `learnings/` 中内容不完整的笔记：

- "是什么"段落为空或只有一句话
- "为什么重要"段落为空
- "代码示例"段落为空
- 没有自测问题

输出：列出不完整的笔记 + 缺少什么

### 3. 过期笔记

扫描掌握度低且长时间未复习的笔记（掌握度定义参考 `protocols/review-system.md`）：

- 掌握度 < 0.4 且超过 14 天未复习
- 建议：要么复习，要么承认"这个暂时不需要"并标记

### 4. 未消化来源

扫描 `sources/` 中状态为"未处理"或"已摘要"的材料：

- 超过 7 天仍未消化的来源
- 建议：要么安排时间学习，要么标记为"暂不学习"

### 5. 日报连续性

检查 `daily/` 目录中是否有断档（工作日没有日报）：

- 列出最近两周中缺失的工作日
- 不强制补写，但提醒用户

## 输出格式

```
知识库健康检查报告：

✅ 总计 X 篇学习笔记，Y 篇问题记录

需要关注：
1. [孤儿笔记] 3 篇笔记没有被其他内容引用：
   - learnings/xxx.md
   - learnings/yyy.md

2. [内容不完整] 2 篇笔记缺少关键段落：
   - learnings/aaa.md — 缺少"代码示例"和"自测问题"

3. [待复习] 4 篇笔记掌握度低且超过一周未复习

4. [未消化来源] 1 个学习材料添加超过 7 天仍未处理

建议优先处理：先补充 learnings/aaa.md 的代码示例（预计 10 分钟）
```

## 修复引导

对于每个问题，提供具体的修复建议：

- 孤儿笔记 → "要不要给 X 添加关联？它和 Y 似乎有关系"
- 空壳笔记 → "要不要现在补充 X 的代码示例？我可以帮你想一个场景"
- 过期笔记 → "要不要快速复习一下 X？只需要 2 分钟"
- 未消化来源 → "要不要现在花 15 分钟看看 X？还是暂时不学了？"

## 不做的事

- 不自动修复（所有修改需要用户确认）
- 不批评用户（"你怎么这么久没复习"）——用建设性语气
- 不一次性列出所有问题（如果问题太多，分批处理，每次最多 3 个）
- 不检查格式/语法错误（那是 lint 脚本的事）
